# 🧮 String Calculator (문자열 계산기)

주어진 문자열 수식을 분석하고 계산하여 결과를 반환하는 Java 기반의 계산기 프로그램입니다.
이 프로젝트는 **TDD(Test-Driven Development, 테스트 주도 개발)** 방법론을 기반으로 작성되었으며, 단순한 덧셈부터 연산자 우선순위, 괄호 처리, 그리고 음수 기호까지 점진적으로 복잡한 수식을 해결하도록 설계되었습니다.

---

## 🏗️ 프로젝트 구조

* **`Calc.java`**: 문자열 수식을 파싱하고 사칙연산을 수행하는 핵심 비즈니스 로직이 담긴 클래스입니다.
* **`CalcTest.java`**: `Calc` 클래스의 기능을 검증하기 위한 28개의 단위 테스트(JUnit 5)가 포함된 테스트 클래스입니다.

---

## 🚀 문제 해결 과정 (TDD 흐름)

테스트 케이스의 번호(`t1` ~ `t28`)가 증가함에 따라, 코드는 다음과 같은 과정을 거쳐 진화했습니다.

* **1단계: 단일 사칙연산 처리 (`t1` ~ `t7`)**
  문자열을 공백 기준으로 나누어 가장 기본적인 덧셈과 뺄셈을 수행하는 구조를 잡았습니다.
* **2단계: 다중 연산자 처리 (`t8` ~ `t11`)**
  연산자가 2개 이상일 때를 대비하여 수식을 재귀적으로 쪼개는(`devide` 메서드) 로직이 도입되었습니다.
* **3단계: 연산자 우선순위 적용 (`t12` ~ `t17`)**
  곱하기(`*`)와 나누기(`/`)가 덧셈/뺄셈보다 먼저 계산되어야 합니다. 이를 위해 수식을 뒤에서부터 탐색하며, 우선순위가 낮은 `+`와 `-`를 먼저 쪼개어 트리의 상단에 위치시키는 방식으로 해결했습니다.
* **4단계: 괄호 연산 도입 (`t18` ~ `t24`)**
  수식에 괄호가 등장합니다. `level` 변수를 사용하여 괄호의 깊이를 추적하고, 괄호로 완전히 둘러싸인 수식은 양끝 괄호를 제거한 뒤 다시 재귀 호출을 하도록 처리했습니다. 괄호 안의 연산자는 분할 대상에서 제외하여 최우선으로 계산되도록 보호했습니다.
* **5단계: 단항 연산자(음수) 및 복합 예외 처리 (`t25` ~ `t28`)**
  이항 연산자(빼기)와 단항 연산자(음수 부호)를 컴퓨터가 구분하지 못하는 문제가 발생했습니다. 연산자 양옆의 공백 여부를 확인하여 빼기와 음수를 구분하고, 괄호 앞의 음수 기호 `-(`는 `-1 * (`로 치환하여 기존 곱셈 로직을 재활용하는 방식으로 우아하게 해결했습니다.

## ⚙️ 핵심 알고리즘 (동작 원리)

`Calc.run()` 메서드가 호출되면, 내부적으로 `devide()` 메서드를 통해 재귀적인 **분할 정복(Divide and Conquer)** 방식으로 수식을 계산합니다.

1. **전처리:** 수식 앞뒤의 공백을 제거하고, 괄호 앞 음수 기호 `-(...`를 `-1 * (...` 형태로 치환합니다.
2. **괄호 벗기기:** 수식 전체가 괄호로 감싸져 있다면(`isWrapped` 검증), 양끝 괄호를 제거합니다.
3. **더하기/빼기 분할:** 괄호 바깥(`level == 0`)에 있고, 양옆에 공백이 있는 `+`나 `-` 연산자를 뒤에서부터 찾습니다. 찾으면 해당 연산자를 기준으로 좌우 수식을 분리하여 재귀 호출합니다.
4. **곱하기/나누기 분할:** 덧셈/뺄셈이 없다면, 동일한 방식으로 `*`나 `/` 연산자를 찾아 분리하고 재귀 호출합니다.
5. **숫자 변환:** 더 이상 쪼갤 연산자가 없는 순수한 숫자 문자열(또는 음수)이 남으면, `Integer.parseInt()`를 통해 정수로 변환하여 반환합니다.

## 💡 주요 기술 포인트

* **재귀 함수 (Recursion):** 복잡한 문자열 수식을 가장 작은 단위(숫자)가 될 때까지 쪼개어 계산하는 데 사용되었습니다.
* **문맥 기반 파싱:** 문자의 단순 매칭이 아닌, 괄호의 `level` 상태와 공백(` `)의 위치라는 문맥을 파악하여 연산자와 단순 기호를 명확하게 구분합니다.
